Bloodlust

Summary

Bloodlust is a game I made with one of my friends for Ludum Dare 49. Ludum Dare is a contest where you are given 72 hours to create a small game based on a particular theme. The theme choosen for this jam was "Prevent the Inevitable" and it was also close to Halloween so we decided to make a game about a vampire who must feast on his victims to survive.

Gameplay

The player starts with a full blood count which starts ticking at the beginning of the game. Once the count reaches zero the player dies. Watch the guard's scheduals to know when the best time to strike will be. But be wary if you let a victim get away they will alert the nearest guard. Some NPCs have weapons which the player can pick up and use to their advantage. The aim of the game is to survive and get as many kills as possible.

Implemented Features

Name Description
Day and Night Cycles Day and night cycle that affects NPC shcedualing and player vision.
Dynamic NPC Scheduals Each NPC is given a specific schedual based on delta time and a waypoint system.
Inventory System An inventory system determines what type of weapon the player has.
AI Reaction Based Behavior NPCs react to the player's attack using a suspician system. This makes them wary of the player if he has attacked them or to go to a guard for help.
